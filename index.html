📋 How to Add New Tools via GitHub

Think of it like a shared recipe book — the HTML file is the book, GitHub is the shelf, and GitHub Pages is the restaurant that serves it. Here's how to add a new "recipe" (tool link):

// STEP 1: Find the SEED_TOOLS array in the HTML file (around line 380+)
// STEP 2: Add a new object to the array, like this:

{
  id: 'seed-2',
  name: 'My New Tool Name',
  url: 'https://example.com/tool-link',
  category: 'Sales Enablement',
  description: 'Short description of what it does.',
  seed: true
}

// STEP 3: Push to GitHub
git add index.html
git commit -m "Added new tool: My New Tool Name"
git push origin main

Categories you can use: Outreach & Prospecting, Sales Enablement, Research & Intelligence, Proposals & Collateral, CRM & Pipeline, Training & Playbooks, Partner Resources, Other

Pro tip: The "Add New Tool" form above saves to the browser's localStorage — great for personal additions. But to make a tool appear for the entire team permanently, add it to the SEED_TOOLS array in the code and push to GitHub. Seeds are the bedrock; localStorage additions are the personal sticky notes.
